Design System

The single source of truth for the Push Films portfolio. Every token, component, and convention documented for consistent design and frictionless development.

01

Principles

01 / Principle
Restraint

Every element earns its place. If removing it doesn't hurt the experience, remove it. The work speaks — the interface should not compete.

02 / Principle
Craft

Micro-details signal intent. Precise letter-spacing, deliberate easing curves, pixel-perfect alignment — these are not finishing touches, they are the point.

03 / Principle
Legibility

Type is the primary interface. Sizes, weights, and colours are chosen for reading comfort at every viewport, not for aesthetic novelty.

04 / Principle
Motion with Purpose

Animation communicates state, not decoration. Every transition has a reason — hover feedback, accordion expansion, gallery advancement — and nothing animates gratuitously.

05 / Principle
Dark by Default

The dark palette is not a mode — it is the design. Film lives in darkness. The near-black canvas maximises image impact and gives the gold accent its full luminance.

Full-Bleed Technique

The work accordion expand panel breaks out of the container using negative margins, not 100vw or transforms. This avoids scrollbar-width miscalculations.

.work-expand { margin-left: calc(var(--pad) * -1); margin-right: calc(var(--pad) * -1); padding-left: var(--pad); padding-right: var(--pad); }

Accessibility

Use <button> element for actions, <a> for navigation. Min touch target 44×44px — padding ensures this. Provide aria-label when label alone is ambiguous. Focus ring must be visible.

Use Primary only for the single most important action per view (showreel, contact, submit).
Don't use more than one Primary button per visible section — it dilutes hierarchy.
.btn-primary { display: inline-flex; align-items: center; gap: 12px; font-family: var(--font-sans); font-size: 10px; letter-spacing: 0.22em; text-transform: uppercase; color: #080808; background: var(--gold); border: 1px solid var(--gold); padding: 16px 32px; cursor: pointer; transition: all 0.3s ease; } .btn-primary:hover { background: #D4B800; border-color: #D4B800; }

Dos & Don'ts

Do

Use gold exclusively for accent and CTA

Gold (#F5D800) signals interactivity. Apply it to active states, buttons, and labels. Its restraint is what makes it powerful.

Don't

Use gold as a decorative colour

Never apply gold to decorative borders, backgrounds, or static text blocks. It will lose its signal value and the hierarchy collapses.

Do

Use DM Sans weight intentionally

Weight 300 for captions and fine detail. Weight 400 for body text and hero sub-headings. Weight 500 for headings, UI labels, and navigation.

Don't

Mix in other typefaces without a strong reason

The single-font system (DM Sans only) is intentional. Its geometric clarity works across display and body sizes — adding a second face risks undermining the clean identity.

Do

Use clamp() for fluid spacing at the page level

--pad and --gap ensure the layout breathes proportionally across every viewport. Use them for section padding and horizontal margins.

Don't

Hard-code px values for page-level spacing

Static pixel values for outer padding create cramped mobile layouts or wasteful desktop ones. clamp() removes the need for most breakpoint overrides.

Do

Animate state changes with purpose

Hover transitions (0.2–0.3s ease), accordion expansion, gallery advancement — all communicate state. Keep durations short and easing natural.

Don't

Add entrance animations to static content

The hero has a purposeful entrance. Other sections do not scroll-animate — they appear immediately. Do not add AOS or scroll-triggered animations; they slow perceived load.

Do

Crop images with object-position: top

The CSS default ensures faces and subjects are visible in the 3:2 thumbnail crop. Override per-image only when subjects are not at the top.

Don't

Use object-position: center as a default

Center crop cuts off heads in portrait-oriented source images. Top is the correct default for this subject matter.

Do

Show gold on every interactive state

Every hovered link, focused input, and active accordion item should resolve to gold. This creates a coherent interaction language.

Don't

Use white or --fg for hover states

White hover states on the dark background lack the warmth and brand presence of gold. --fg is reserved for primary content, not interaction feedback.

Do

Use the negative-margin full-bleed technique

The accordion expand panel uses calc(var(--pad) * -1) margins to break out of the container cleanly, without 100vw scrollbar issues.

Don't

Use width:100vw for full-bleed sections

100vw includes the scrollbar width on Windows, causing horizontal overflow. The negative-margin technique is reliable across all browsers and OS.

Do

Put mobile accordion overrides in the second media block

The accordion desktop CSS sits after the first mobile block in the stylesheet. Equal-specificity rules later in the file win — so always use the second !important block at end of <style>.

Don't

Add mobile accordion overrides in the first @media block

They will be silently overridden by the desktop accordion rules that follow later in the cascade, with no error or warning.

Do

Detect iOS before setting Vimeo autoplay

iOS Safari permanently blocks cross-origin iframe autoplay. The isIOS check omits autoplay=1 on those devices, preventing an infinite spinner.

Don't

Add controls=0, vimeoPlayer.play(), or setMuted() on iOS

All three cause an infinite loading spinner on iOS Safari for cross-origin Vimeo iframes. Leave iOS to use Vimeo's native play button.

Do

Quote filenames with spaces in git add

git add "images/EWR (12).jpg" — the quotes are essential. Without them the shell interprets the space as argument separation and the file is not staged.

Don't

Commit large video files without checking size

Check with du -m before staging. GitHub hard limit is 100MB per file; warning at 50MB. MOV files from iPhone can easily exceed 100MB.

Do

URL-encode filenames with spaces in HTML src attributes

images/EWR (12).jpg → images/EWR%20(12).jpg. Spaces in src paths break image loading silently in some browsers.

Don't

Use HEIC files as image sources

HEIC is an Apple-proprietary format not supported by browsers. Always use the .jpg equivalent. Convert before adding to the project.

# Python helper to apply position exports to gallery items # (run in project root, pass the copied JSON as input) import json, re, sys positions = json.loads(sys.stdin.read()) # positions = { "raptor-2": "50% 30%", "everest-5": "50% 70%" } # Then manually apply each to the matching gallery item: # galleries.raptor.items[2].position = "50% 30%" # galleries.everest.items[5].position = "50% 70%"

Mobile Override Pattern

The accordion desktop CSS sits at ~line 590 in index.html, after the first @media (max-width: 640px) block at ~line 558. Equal-specificity rules later in the file win, so any mobile accordion overrides in that first block are silently ignored.

/* ✕ WRONG — overridden by later desktop accordion rules */ @media (max-width: 640px) { /* ~line 558 — FIRST block */ .expand-film { width: 100%; } /* silently ignored */ } .expand-film { width: 87.5%; } /* ~line 590 — wins due to cascade order */ /* ✓ CORRECT — second @media block at very end of <style> */ @media (max-width: 640px) { /* END of <style> */ .expand-film { width: 100% !important; padding-bottom: 56.25% !important; } }
